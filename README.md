# Project Overview

Sales analytics is a key part of modern data-driven business decision making.

This project demonstrates a complete sales analytics workflow using SQL and Power BI.  
It simulates a real business scenario where analysts explore sales data to understand trends, evaluate product performance, and analyze customer behavior.

The project includes:

- database design
- synthetic data generation
- analytical SQL queries
- business metrics analysis
- Power BI dashboard

Sales analytics is commonly used in:

- e-commerce companies
- retail analytics
- SaaS analytics
- financial reporting
- business intelligence teams

---

# Business Problem

Companies generate large amounts of sales data, but without proper analysis it is difficult to extract meaningful insights.

Businesses often struggle to answer questions such as:

- Which products generate the most revenue?
- How do sales change over time?
- Who are the most valuable customers?
- What products are frequently purchased together?

Without proper analytics, companies risk:

- making decisions based on intuition instead of data
- missing opportunities for revenue growth
- failing to identify top-performing products
- misunderstanding customer behavior

Sales analytics helps organizations make data-driven decisions that improve performance and profitability.

# Database Model

The project simulates a simplified sales data warehouse.

Main transactional tables:

- customers
- products
- orders
- order_items

Analytical tables:

- fact_sales
- dim_customer
- dim_product
- dim_date

The structure follows a simplified star schema, which is commonly used in business intelligence systems.

# Example SQL Query

Example query calculating monthly revenue.
SELECT
    DATE_TRUNC('month', order_date) AS month,
    SUM(revenue) AS total_revenue
FROM fact_sales
GROUP BY 1
ORDER BY 1;

# How to Run the Project

Clone the repository:
git clone https://github.com/T0r0s68/sql-sales-analytics-project.git

Create a PostgreSQL database.

Run the SQL scripts in order:
01_create_tables.sql
02_generate_data.sql
08_data_warehouse.sql

Run analytical queries if needed:
03_sales_analysis.sql
04_customer_analysis.sql
05_product_analysis.sql
06_cohort_analysis.sql
07_market_basket.sql
09_advanced_analysis.sql

Connect Power BI to the PostgreSQL database and create the dashboard.
